# skill: token-saving
Objetivo: Minimizar tokens sin perder calidad de trabajo.
Cuándo: Siempre. Especialmente en sesiones largas con contexto acumulado.
Checklist:
- Lee solo archivos relacionados con el P0 actual
- No repitas contexto en respuestas (el usuario ya lo sabe)
- No pegues código completo si puedes mostrar solo el diff
- No hagas auditorías completas si ya hay BUG_BACKLOG.md actualizado
- Usa grep/find antes de leer archivos enteros
- Actualiza los .md de estado en lugar de explicarlo en chat
Output: Cambios mínimos, descripciones máximas 20 líneas.
